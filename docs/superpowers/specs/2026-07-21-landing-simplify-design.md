# Super MCP Landing Simplification

## Goal
Make the marketing page understandable in five seconds: a shopping list becomes a reliable store plan. Technical proof stays available through progressive disclosure.

## Message
**Primary:** שולחים רשימת קניות. מקבלים תוכנית קנייה חכמה.  
**Support:** Super MCP מוצא מוצרים נכונים, משווה מחירים ומחזיר תוכנית שאפשר לסמוך עליה.

## Page structure
1. Hero — outcome + CTA + grocery visual (no API panel)
2. Three benefits — right product, real prices, asks when unsure
3. How it works — list → clarify → plan
4. One example — single recommended store result in plain Hebrew
5. Trust — asks before silent substitution
6. Access CTA — request access; manual config collapsed
7. Developer disclosure — tools collapsed by default
8. Transparency footer

## Explicit removals
- Technical hero dossier
- Six reliability cards
- Three competing plan cards
- Standalone freshness band
- Visible API field names before the developer section

## Visual
- Wolt cyan palette with darker CTA for AA contrast
- One primary CTA: בקשו גישה
- Cards reserved for the example and access panel
